Events editor
Overrides are applied on the next preprocess run.
Read-only for this campaign. This campaign's events.json is owned by the GH-Actions Discord sync, which doesn't read overrides.json. You can browse parsed and untagged events here, but edits won't take effect on the deployed site. To change tags / fix typos, edit the underlying Discord posts (or extend sync_events.py).
When the file picker opens, navigate to …/data/overrides.json and save there. The path is relative to your repo root; the file replaces (or creates) overrides.json inside this campaign's data/ folder.

Edits go into data/overrides.json; the next preprocess run merges them back onto the parsed event. Leave a field blank to keep the parser's value.

Messages the parser couldn't promote to events. Click Promote to event to manually flag one; the next preprocess run will lift it into events.json.
